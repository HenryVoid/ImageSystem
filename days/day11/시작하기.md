# Day 11: 빠른 시작 가이드

> 5분 안에 이미지 압축 효율 실험 학습 시작하기

---

## 🚀 빠른 시작

### 1. 프로젝트 열기 (30초)

```bash
cd day11
open day11.xcodeproj
```

Xcode가 열리면 자동으로 SDWebImageWebPCoder 패키지를 다운로드합니다.

---

### 2. 앱 실행 (10초)

```
⌘R (Run)
```

시뮬레이터에서 앱이 실행됩니다.

---

### 3. 첫 실험 (3분)

#### Step 1: 이미지 로드 (30초)

1. **인터랙티브** 탭 클릭
2. "샘플 이미지 로드" 버튼 클릭
3. 이미지 표시 확인

**체험한 것**: 기본 샘플 이미지 로드

---

#### Step 2: JPEG 압축 (1분)

1. 포맷: **JPEG** 선택 (기본값)
2. 품질: **80%** 설정
3. "압축 실행" 버튼 클릭
4. 결과 확인:
   - 원본 크기: ~3MB
   - 압축 후: ~1MB
   - 압축률: ~70%
   - 시간: ~20ms

**체험한 것**: 
- 70% 크기 감소 📉
- 20ms 빠른 처리 ⚡
- 품질은 육안으로 거의 차이 없음 ✅

---

#### Step 3: 품질 변경 (30초)

1. 품질 슬라이더를 **50%**로 조정
2. "압축 실행" 클릭
3. 결과 비교:
   - 크기: 더 작아짐 (~500KB)
   - 품질: 눈에 띄게 저하 ⚠️

**체험한 것**: 품질-크기 트레이드오프

---

#### Step 4: 포맷 비교 (1분)

1. **HEIC** 포맷 선택
2. 품질: **80%** 설정
3. "압축 실행" 클릭
4. JPEG 결과와 비교:
   - HEIC가 약 50% 더 작음! 🎯
   - 하지만 2-3배 느림

**체험한 것**: HEIC의 압축 효율

## 🎯 5가지 핵심 실험

### 실험 1: JPEG 품질 체험 (2분)

1. 인터랙티브 탭 → 샘플 이미지 로드
2. JPEG, 품질 100% → 압축 (약 3MB)
3. JPEG, 품질 80% → 압축 (약 1MB) ✅
4. JPEG, 품질 50% → 압축 (약 500KB) ⚠️

**배운 것**: 
- 80%가 최적점 (Sweet spot)
- 50%는 품질 저하 심함

---

### 실험 2: 포맷 효율 비교 (3분)

1. 벤치마크 탭 → 이미지 로드
2. "벤치마크 시작" 클릭 → 대기 (30초)
3. 결과 확인:

```
크기 (작은 순):
1. HEIC:  1.2 MB ⭐⭐⭐
2. WebP:  1.8 MB ⭐⭐
3. JPEG:  2.5 MB ⭐
4. PNG:   7.5 MB

속도 (빠른 순):
1. JPEG:  18 ms ⚡⚡⚡
2. WebP:  30 ms ⚡⚡
3. HEIC:  45 ms ⚡
4. PNG:  120 ms
```

**배운 것**: 
- HEIC가 최고 효율
- JPEG가 최고 속도

---

### 실험 3: 차트 분석 (2분)

1. 비교 탭 → 이미지 로드
2. "비교 데이터 생성" 클릭
3. 메트릭 변경:
   - 파일 크기 → HEIC가 최소
   - 압축률 → HEIC가 최고
   - 처리 시간 → JPEG가 최고

**배운 것**: 시각적으로 차이 명확히 확인

---

### 실험 4: 해상도별 차이 (3분)

1. 이미지 탭 → 해상도 "작게 (300×300)" → 다운로드
2. 인터랙티브 탭 → JPEG 80% 압축
   - 결과: ~50KB
3. 이미지 탭 → 해상도 "크게 (1920×1080)" → 다운로드
4. 인터랙티브 탭 → JPEG 80% 압축
   - 결과: ~1.5MB

**배운 것**: 
- 해상도가 크기에 큰 영향
- 다운샘플링의 중요성

---

### 실험 5: 실전 시나리오 (5분)

#### 웹 이미지 최적화
```

```

**배운 것**: 용도에 맞는 포맷과 품질 선택

## 💡 문제 해결

### Q: 패키지 다운로드가 안 됨
**A**: 
1. Xcode 재시작
2. File → Packages → Reset Package Caches
3. 네트워크 연결 확인
4. 다시 빌드

---

### Q: HEIC 압축이 실패함
**A**:
1. iOS 11+ 시뮬레이터 사용 확인
2. Info.plist 설정 확인
3. 다른 포맷으로 테스트

---

### Q: 이미지가 다운로드되지 않음
**A**:
1. 인터넷 연결 확인
2. Safari에서 `https://picsum.photos/300/300` 테스트
3. 샘플 이미지 사용 (오프라인 가능)

---

### Q: 압축이 너무 느림
**A**:
1. 해상도 "작게" 또는 "중간" 사용
2. Release 모드로 빌드 (⌘B)
3. 실제 기기에서 테스트

---

### Q: 품질 차이를 모르겠음
**A**:
1. 해상도 "크게"로 다운로드
2. 품질 100% vs 50% 극단 비교
3. 이미지를 확대해서 확인
