## ⚡ 빠른 시작 (5분)

### 1. 프로젝트 열기 (30초)

```bash
cd day14
open day14.xcodeproj
```

### 2. 빌드 & 실행 (2분)

```
⌘R (Run)
```

첫 실행 시 SPM 패키지(Nuke, Kingfisher) 다운로드로 약 1-2분 소요됩니다.

### 3. 네트워크 연결 확인 (10초)

이미지 로딩을 위해 인터넷 연결이 필요합니다.
Wi-Fi 또는 모바일 데이터 연결을 확인하세요.

### 4. 앱 탐색 (2분)

5개 탭을 순서대로 둘러보세요:
1. **그리드** 📷: 썸네일 갤러리
2. **리스트** 📋: 상세 정보 리스트
3. **비교** ⚖️: Nuke vs Kingfisher
4. **검색** 🔍: 검색 & 필터
5. **성능** 📊: 실시간 통계

## 🎯 핵심 기능 5분 체험

### 시나리오 1: 그리드 갤러리 (1분)

1. **그리드** 탭 선택
2. 200개 이미지 썸네일 로딩 관찰 (약 30초)
3. 스크롤하며 이미지 탐색
4. 썸네일 탭 → 상세보기 이동
5. 다시 위로 스크롤 → **즉시 표시** (캐시!)

**학습 포인트**: 캐시로 60배 빠른 재로드

---

### 시나리오 2: 라이브러리 비교 (2분)

1. **비교** 탭 선택
2. 이미지 개수: **50개** 선택
3. **"비교 시작"** 버튼 클릭
4. Nuke(왼쪽) vs Kingfisher(오른쪽) 동시 로딩
5. 통계 확인:
   - 캐시 히트율
   - 평균 로드 시간
   - 메모리 사용량
6. 분석 결과 확인

**학습 포인트**: 라이브러리 성능 차이 실측

---

### 시나리오 3: 검색 & 필터 (1분)

1. **검색** 탭 선택
2. 검색 바에 작가명 입력 (예: "Paul")
3. 결과 즉시 필터링 확인
4. 카테고리 필터 선택 (예: "큰 이미지")
5. **북마크** 칩 탭
6. 이미지 탭 → 상세보기 → 북마크 토글
7. 검색 탭으로 돌아가기 → 북마크된 이미지만 표시

**학습 포인트**: 실시간 필터링 및 북마크

---

### 시나리오 4: 상세보기 (30초)

1. 아무 탭에서 이미지 선택
2. 상세보기 진입
3. 이미지 더블 탭 → **2배 줌**
4. 핀치 줌 → **5배까지 확대**
5. 드래그하여 이동
6. 좋아요/북마크 토글
7. **"공유"** 버튼 → 이미지 공유

**학습 포인트**: 줌, 공유, 인터랙션

---

### 시나리오 5: 성능 모니터 (30초)

1. **성능** 탭 선택
2. **Nuke** 선택
3. Nuke 통계 확인:
   - 로드 횟수
   - 캐시 히트율
   - 메모리 사용량
4. **Kingfisher** 선택 → 통계 비교
5. 시스템 메모리 확인
6. **"분석"** 버튼 → 비교 결과

**학습 포인트**: 실시간 성능 모니터링

## 🔧 문제 해결

### Q1: 이미지가 로드되지 않아요

**확인사항**:
- [ ] 인터넷 연결 확인
- [ ] Picsum Photos API 접근 가능 확인
- [ ] 앱 재시작

**해결책**:
```swift
// ImageProvider.swift의 baseURL 확인
private let baseURL = "https://picsum.photos/v2/list"
```

---

### Q2: 빌드 에러가 발생해요

**SPM 패키지 문제**:
```bash
# Xcode에서
File → Packages → Reset Package Caches
File → Packages → Update to Latest Package Versions
```

**빌드 정리**:
```
⇧⌘K (Clean Build Folder)
⌘B (Build)
```

---

### Q3: 메모리 경고가 발생해요

**원인**:
- 시뮬레이터에서 200개 이미지 로드 시 발생 가능

**해결책**:
1. 캐시 삭제 (⋯ 메뉴)
2. 앱 재시작
3. 이미지 개수 줄이기 (ImageProvider에서 페이지 수 조절)

---

### Q4: 성능이 느려요

**최적화 확인**:
- [ ] 다운샘플링 적용됨 (`thumbnailURL(size: 300)`)
- [ ] 캐시 활성화됨
- [ ] Lazy 로딩 사용됨

**개선 방법**:
```swift
// NukeImageLoader.swift에서 캐시 크기 조절
imageCache.costLimit = 100 * 1024 * 1024 // 100MB
```
