# Day 10: 빠른 시작 가이드

> 5분 안에 이미지 캐싱 전략 학습 시작하기

---

## 🚀 빠른 시작

### 1. 프로젝트 열기 (30초)

```bash
cd day10
open day10.xcodeproj
```

Xcode가 열리면 자동으로 Kingfisher와 Nuke 패키지를 다운로드합니다.

---

### 2. 앱 실행 (10초)

```
⌘R (Run)
```

시뮬레이터에서 앱이 실행됩니다.

---

### 3. 첫 실험 (3분)

#### Step 1: 캐시 효과 체험

1. **Kingfisher** 탭 클릭
2. "이미지 로드" 관찰 → 약 30초 소요
3. **"다시 로드"** 버튼 클릭
4. 즉시 표시! (0.5초) ⚡

**체험한 것**: 캐시가 이미지를 **60배 빠르게** 로드

---

#### Step 2: 통계 확인

화면 상단에서 확인:
- **메모리 캐시**: ~50MB
- **히트율**: 95%+
- **로드된 이미지**: 100/100

**체험한 것**: 실시간 캐시 통계

---

#### Step 3: 프리히팅 체험

1. **Nuke** 탭으로 이동
2. **"캐시 삭제"** 클릭
3. **"프리히팅"** 버튼 클릭 → 3초 대기
4. 스크롤 → 모든 이미지 즉시 표시!

**체험한 것**: 프리히팅으로 완벽한 UX

---

## 📱 4개 탭 소개

### Tab 1: Kingfisher 갤러리
```
기능: 100개 이미지 로딩 + 캐시 통계
핵심: 캐시 효과를 직접 체험
```

**주요 버튼**:
- 다시 로드: 캐시 효과 확인
- 캐시 삭제: 초기화
- 통계 초기화: 히트율 리셋

---

### Tab 2: Nuke 갤러리
```
기능: Kingfisher와 동일 + 프리히팅
핵심: Nuke의 고급 기능 체험
```

**차별점**:
- 프리히팅 버튼
- 약간 더 빠른 성능
- 더 높은 히트율

---

### Tab 3: 캐시 모니터
```
기능: 실시간 캐시 상태 모니터링
핵심: 상세한 통계와 비교
```

**4개 서브 탭**:
- Kingfisher 모니터
- Nuke 모니터
- 시스템 모니터
- 비교 뷰

---

### Tab 4: 캐시 설정
```
기능: 캐시 정책 설정
핵심: 다양한 전략 실험
```

**프리셋**:
- 최소 (30MB / 100MB)
- 균형 (100MB / 500MB) ← 권장
- 공격적 (200MB / 1GB)
- 커스텀 (자유 설정)

## 🎯 5가지 핵심 실험

### 실험 1: 캐시 없음 vs 있음 (1분)

1. Kingfisher 탭 → 이미지 로드 (30초)
2. "다시 로드" (0.5초)

**배운 것**: 캐시가 60배 빠름

---

### 실험 2: Kingfisher vs Nuke (2분)

1. Kingfisher 탭 → 로드 + 시간 확인
2. Nuke 탭 → 로드 + 시간 확인
3. 캐시 모니터 → 비교 탭

**배운 것**: Nuke가 약간 더 빠름

---

### 실험 3: 프리히팅 효과 (1분)

1. Nuke 탭 → "캐시 삭제"
2. "프리히팅" 클릭
3. 스크롤 → 즉시 표시

**배운 것**: 프리히팅으로 완벽한 UX

---

### 실험 4: 메모리 부족 시뮬레이션 (3분)

1. 설정 탭 → 커스텀
2. 메모리 캐시: 10MB (매우 작게)
3. 설정 적용
4. Kingfisher 탭 → 200개 로드
5. 캐시 모니터 → 메모리 사용량 관찰

**배운 것**: LRU가 자동으로 오래된 이미지 삭제

---

### 실험 5: TTL 정책 (5분)

1. 설정 탭 → 커스텀
2. TTL: 5분
3. 설정 적용
4. 이미지 로드
5. **5분 대기** (다른 일 하기)
6. 다시 로드

**배운 것**: TTL 만료 후 다시 다운로드

## 💡 문제 해결

### Q: 패키지 다운로드가 안 됨
**A**: 
1. Xcode 재시작
2. File → Packages → Reset Package Caches
3. 다시 빌드

---

### Q: 이미지가 로드되지 않음
**A**:
1. 인터넷 연결 확인
2. 시뮬레이터 네트워크 설정 확인
3. Safari에서 `https://picsum.photos/300/300` 접속 테스트

---

### Q: 캐시가 동작하지 않는 것 같음
**A**:
1. 캐시 모니터 탭 → 통계 확인
2. "통계 초기화" → 다시 테스트
3. 히트율이 0%면 캐시 설정 확인

---

### Q: 앱이 느려짐
**A**:
1. 캐시 모니터 → 시스템 탭
2. 메모리 사용량 확인
3. "캐시 삭제" 버튼 클릭
